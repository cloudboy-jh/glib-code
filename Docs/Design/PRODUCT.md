# Product

## Register

product

## Users
Primary users are solo developers running local repositories with an AI pair-programming loop. They are usually in active coding sessions, switching between repo context, session context, and settings while trying to keep momentum. Their job is to configure providers, model behavior, Git/GitTrix behavior, and interface preferences fast, then get back to shipping code.

## Product Purpose
glib-code is an AI coding workspace that keeps agent edits isolated until the user explicitly promotes accepted changes. It exists to give developers speed without sacrificing control. Success means users can open a repo, configure runtime/settings confidently, run focused sessions, review diffs clearly, and promote only what they trust.

The product now includes core durable-repo git workflow controls (stage/unstage/discard/commit/pull/checkout/branch creation and commit inspection) directly in-app.

## Brand Personality
Calm, surgical, trustworthy.

Voice and tone should feel direct and competent, never fluffy, never hype-heavy. The interface should signal expert control, low noise, and operational confidence. Emotionally, the product should evoke clarity and trust while users make high-impact decisions.

## Anti-references
- Generic SaaS template dashboards, especially repeated card-grid UI patterns with shallow hierarchy.
- Flashy neon “hacker tool” aesthetics with decorative animation and novelty over clarity.
- Heavy enterprise admin interfaces with bloated forms, excessive ceremony, and bureaucratic friction.

## Design Principles
1. Isolate first, promote deliberately.
2. Make important decisions obvious, make routine actions frictionless.
3. Prefer calm density over decorative complexity.
4. Keep advanced control visible without overwhelming primary workflows.
5. Every settings surface should reduce uncertainty, not add explanation debt.

## Accessibility & Inclusion
- Baseline target: WCAG 2.1 AA.
- Maintain strong contrast and keyboard-first operability across settings and session-critical controls.
- Respect user reduced-motion preferences by removing non-essential animation and preserving clarity of state changes.
