## Part 1 — Terminal (fix + cleanup) — DONE

### What it was
- `server/src/routes/term.ts` was a **command runner over a websocket**, not a PTY terminal. Each `run` spawned a fresh `shell -lc "cmd"`, buffered all stdout/stderr until exit (`new Response(proc.stdout).text()`), then sent one `output` frame per stream plus an `exit`. No streaming, no cancel, no persistent shell state, no process registry. `onClose` was a no-op (orphans children).
- `web/src/components/app/TerminalDrawer.vue` was a textarea + Run button + single growing `<pre>`. Status pill read a `status` prop. No history, no clear, no per-command separation.
- **Bug:** the drawer never sent `cwd`, so commands ran in the server's `process.cwd()` instead of the active project. This contradicted the strict-project-scope work in session 3a.

### What was done
1. **Scope to the project.** The drawer sends the active project's path as `cwd` on every `run`. The server resolves cwd via `data.cwd` → `fallbackProjectPath()` → `NO_PROJECT` error. No silent server-cwd fallback. Terminal disabled in the UI when no project is open.
2. **Stream output.** `proc.stdout` / `proc.stderr` chunks piped as `output` frames as they arrive via `ReadableStream` readers with `TextDecoder({ stream: true })`. `exit` frame sent on completion.
3. **Cancel.** Per-socket process registry keyed by `run` id. Inbound `{ type: "cancel"; id }` frame kills the process (SIGTERM → SIGKILL after 1.5s grace). On websocket close, all live processes for that socket are SIGKILLed — `onClose` no-op replaced.
4. **Frontend cleanup.** Transcript UI with per-command entries (`$ command` header + streaming output body + status chip). Clear button. Up/down arrow history recall. Honest "Command runner" label + `$` prompt prefix + contextual placeholders. Fixed-height drawer (38vh) with transcript as dominant region. Connection dot with aria labels. `role="log"` + `aria-live="polite"` on transcript. Auto-scroll respects user scroll position.
5. **Honest capabilities.** `ready` frame extended: `capabilities: { run: true, interactive: false, cancel: true, shell, platform }`.
6. **Persistent cwd skipped** — not cheap (PTY-adjacent or fragile parsing), explicitly a non-goal.

### Bonus fix
- `server/src/index.ts`: `Bun.serve` was missing the `websocket` handler from `hono/bun` — WS connections would have failed in production. Fixed by passing `websocket: honoWebSocket` to `Bun.serve`.

### Tests
- `server/src/routes/term.test.ts` (new, 11 tests): ready capabilities, cwd scoping (fallback/explicit/no-project), streaming frame order, cancel kill, cancel not-found, socket-close orphan cleanup, empty command, hello/ack, ping/pong.

### NON-GOALS (respected)
- No PTY / no real terminal emulator. No `node-pty`, no `xterm.js`, no native modules.
- No interactive programs (vim, top, REPLs, prompts that read stdin mid-run).
- No multi-tab / multiple concurrent terminals.
