# UI/UX Feature List - 2026-05-10

This list is grounded against the current `glib-code` codebase, docs, and server routes. Core workflow stays unchanged:

1. Open a git repo.
2. Review commit or uncommitted diffs.
3. Start a session with selected diff context.
4. Chat with an agent running in a GitTrix ephemeral workspace.
5. Promote accepted changes back to the durable repo.

## Current Backend Reality

- GitTrix already models durable repo plus ephemeral workspace.
- Sessions persist `gittrixSessionId`, `ephemeralPath`, and `baselineSha`.
- Agent turns run in the session ephemeral workspace.
- `/api/git/status`, `/api/git/branches`, and `/api/git/log` exist.
- `/api/diff/hunks` exists.
- Settings already support durable provider, ephemeral provider, and promote strategy.
- Git mutation routes are still stubbed: stage, unstage, discard, commit, push, pull, checkout, create branch, commit detail.
- `/api/diff/branch-compare` is still stubbed.
- `/api/term` and `/api/attachments` are planned but not fully wired in the frontend.

## P0

- Add hunk-level context selection using `/api/diff/hunks`.
- Replace the current stub-style `HunkList.vue` and `SelectionTray.vue` with app-token styled production components.
- Add a multi-select tray for selected files and hunks before starting or continuing a session.
- Keep selected diff context visible in the composer as removable chips.
- Add diff stats/counts everywhere they help scanning: changed file count, selected file count, hunk count, selected hunk count, additions, deletions, and truncation count.
- Add diff toolbar polish: sticky controls, split/unified toggle, file nav, file picker, truncation warning, loading states, and error states.
- Upgrade tool-call cards so tool execution is understandable without dumping raw logs.
- Add deterministic keyboard handling for picker, command palette, composer commands, diff file nav, and overlays.
- Fix overlay behavior: top-most `Esc` only, click-outside consistency, modal stacking, and focus return.
- Add a lightweight workspace toolbar showing current branch, durable provider, ephemeral provider, promote strategy, baseline/session workspace state, and GitTrix settings access.

## P1

- Upgrade composer command UX: better slash search, grouped commands, keyboard navigation, command descriptions, and disabled reasons.
- Add composer status banners for missing provider key, unavailable model, GitTrix startup failure, missing Cloudflare config, and backend disconnected states.
- Improve provider/model picker with usable-only filter, provider grouping, unavailable reasons, and inline auth actions.
- Add project-level provider/model override UI with effective state labels.
- Improve promote modal with selected file/hunk summary, clearer baseline conflict copy, baseline drift metadata, and explicit "what will be promoted" text.
- Add timeline collapse for long turns, summaries, and tool-call groups.
- Add timeline/session status indicators: running, stopped, errored, waiting, and streaming.
- Improve session sidebar with resize persistence, collapsed tooltips, active project/session clarity, and skeleton/loading states.
- Add Git action panel/state: show implemented status/log/branches now, but keep mutation actions disabled with explicit route/support reasons.

## P2

- Replace mock terminal drawer with real `/api/term` WebSocket UI once backend transport is ready.
- Add terminal reconnect/disconnected states.
- Route terminal cwd through active project or active GitTrix ephemeral workspace depending on context.
- Add attachments UI once `/api/attachments` is ready: file/image picker, preview chips, size errors, upload errors, and removable attachment chips.
- Add compact readiness/health surface showing git, gh, pi/provider readiness, Cloudflare config, terminal route availability, and attachment route availability.
- Add artifact/right-side preview panel only if sessions produce saved outputs worth previewing.
- Add branch/worktree/environment controls only after git mutation routes and branch compare are real.

## Branch / Workspace Toolbar Scope

Do now:

- Show current branch.
- Show durable provider: `local` or `github`.
- Show ephemeral provider: `local` or `cloudflare-artifacts`.
- Show promote strategy: `commit`, `branch`, `pr`, or `patch`.
- Show baseline SHA/session workspace metadata where useful.
- Link to GitTrix settings.
- Display unsupported branch mutation controls as disabled with reasons if they are visible at all.

Wait until backend support lands:

- Branch checkout.
- Create branch.
- Compare branch.
- Worktree/environment switching.
- Pull and push controls.
- PR creation/opening as a primary flow.
- Any T3-style environment mode selector that implies live switching.

## Diff UX

- Do not advertise `branches` and `prs` as active sources unless implemented.
- If visible, show `branches` and `prs` disabled with clear copy: "Requires branch compare/source-control support."
- Keep commit and working tree diff flows first-class.
- Show compact count metadata in the diff header: total files, total hunks, additions, deletions, selected files, selected hunks.
- Show per-file metadata in the file picker/list: status, additions/deletions if available, hunk count, selected hunk count.
- Show source metadata for open diffs: working tree, commit SHA, commit title, author/date if available.
- Make file navigation stable for large diffs.
- Keep truncation copy explicit when `MAX_PATCH_LINES` is hit.
- Reuse the same selection primitive for session context and promote selection.

## Tool-Call UX

- Keep tool calls compact by default; expand only the active/running call automatically.
- Group tool calls by turn so users can scan "what the agent did" without reading every event.
- Show clear status per tool: queued, running, succeeded, failed, cancelled.
- Show elapsed time for long-running tools.
- Show tool name, target path/command, short result summary, and error summary before any raw details.
- Hide raw JSON behind an "Inspect details" disclosure.
- For file tools, show affected file path, operation type, and a quick jump to diff/context where possible.
- For shell tools, show command, cwd, exit code, duration, and last useful output lines.
- For failed tools, show the failure in the timeline and composer banner if it blocks the next user action.
- Let users collapse/expand all tool calls in a turn.
- Avoid making tool cards look like assistant prose; they should read as structured execution records.

## Counts / Status Indicators

- Header/session status: active model, provider auth state, branch, workspace type, running/stopped/error state.
- Sidebar session rows: status dot, last updated, provider/model short label, and dirty/session-diff indicator if available.
- Diff mode: changed files count, selected context count, current file index, total hunks, selected hunks.
- Composer: attached context count, attachment count, command mode indicator, send disabled reason.
- Promote modal: selected files/hunks count, total changed files, baseline SHA, conflict count if present.
- Settings: providers connected count, usable models count, GitTrix provider readiness count.
- Readiness surface: git, gh, pi/provider, terminal, attachments, Cloudflare config status.

## Composer UX

- Preserve the current simple textarea flow.
- Add context chips above or inside the composer footer.
- Chips should represent selected files, hunks, terminal snippets, and attachments as those become available.
- Commands should be keyboard-first and should not require mouse interaction.
- Commands that cannot run should remain visible only when useful, with disabled reasons.
- Missing capability states should appear as banners near the composer, not as silent send failures.

## Settings UX

- GitTrix settings should explain durable vs ephemeral in plain product language.
- Cloudflare Artifacts should show missing environment/token requirements clearly.
- GitHub durable should show auth state and origin remote requirements.
- Provider/model settings should distinguish global defaults from project overrides.
- Keybindings editor should reflect backend rules and command availability.

## Empty States

- Project picker: no recents, missing path, missing git, provider not configured.
- Diff: no commits, clean worktree, no file selected, branch/PR unsupported.
- Session: no active session, provider key missing, model unavailable, GitTrix setup failed.
- Promote: no changes, no files selected, baseline conflict.
- Terminal/attachments: unavailable route, disconnected route, upload failure.

## Visual / Component Cleanup

- Remove one-off hardcoded colors where design tokens exist.
- Bring old/stub components into the same visual system as current cards, dialogs, and composer.
- Keep the workbench quiet, dense, and native-feeling.
- Avoid large decorative UI moments inside workflow surfaces.
- Keep responsive behavior good enough for picker, settings, model picker, and dialogs on smaller widths.

## Suggested Build Order

1. Diff hunk/file selection and selection tray.
2. Composer context chips and command UX.
3. Lightweight workspace toolbar.
4. Overlay and keybinding correctness.
5. Diff/promote polish.
6. Provider/model/settings polish.
7. Timeline and session sidebar polish.
8. Terminal and attachments after backend routes are ready.
